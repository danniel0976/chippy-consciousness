# March 17, 2026 - TWYLM Website Fixes

## Session Summary
Fixed multiple issues with TWYLM calendar and user management.

## Key Issues Fixed

### 1. Luke User Identification
**Problem:** Luke's entries (`donuteatsalot@gmail.com`) were overwriting Dan's entries because:
- Luke had no record in `users` table
- Code identified users by `name` only, which returned empty for Luke
- Luke's entries got stored as `date` instead of `date_luke`

**Solution:** 
- Identify Luke by email fallback: `userEmail === 'donuteatsalot@gmail.com'`
- For production: identify by `userName.toLowerCase() === 'luke'`
- Store Luke's entries as `date_luke` key, Dan's as `date`
- Tile shows Dan's entry (priority), Luke's accessible via hover bubble

**Luke's user_id:** `1e4d0604-2cdb-4c32-8c09-072360a98638` (found via database query)

### 2. Calendar Hover Bubble
**Problem:** Bubble disappeared when user tried to click it

**Solution:**
- Added 300ms delay before hiding bubble
- Made bubble itself hoverable (shares mouse handlers with tile)
- Both tile and bubble use `onMouseEnter`/`onMouseLeave` with clearTimeout

### 3. Cheese Mascot Clipping
**Problem:** 
- Mobile: Cheese clipped at screen edges
- Desktop: Header menu expanded when Cheese walked right

**Solution:**
- Mobile: Container uses `-mx-4` to bleed beyond viewport (reverted to normal page wrapper)
- Desktop: Added `overflow-x-hidden` to page wrapper instead of container margin
- Cheese walks 0-100% width, page clips overflow not header

### 4. Delete Account Function
**Added to Profile page:**
- Double confirmation dialogs (permanent action warning)
- Deletes: diary_entries → users table → auth user
- Redirects to home after completion
- Red "Danger Zone" button styling
